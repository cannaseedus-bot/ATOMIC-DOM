Atomic Blocks: All-or-Nothing Execution

Atomic Blocks guarantee that all DOM mutations within a block either complete together or not at all. This eliminates partial updates and ensures your UI is always in a consistent state.

example.asxr 
@atomic [dom-update] {{ '{' }}
  // All changes batched into ONE reflow
  element.style.width = '100px';
  element.style.height = '200px';
  element.style.margin = '10px';
  element.classList.add('active');
{{ '}' }}

Benefits

  • Single layout/reflow per block
  • Automatic rollback on errors
  • Predictable state transitions
  • Easier debugging and testing

Use Cases

  • Complex UI state changes
  • Form validation updates
  • Animation sequences
  • Data-driven rendering

State Synchronization: Reactive Bindings

Connect application state to DOM state with reactive bindings that maintain consistency. Changes flow automatically while preserving atomic guarantees.

sync.asxr 
@state [user] {{ '{' }}
  name: "John",
  theme: "dark"
{{ '}' }}

@atomic [header] {{ '{' }}
  @dom h1 {{ '{' }}
    text: {{ '{{user.name}}' }}
  {{ '}' }}
{{ '}' }}
